Android 4.4 以上沉浸式状态栏实现

这几天看了很多关于沉浸式状态栏的实现方案，总的在此汇总整理一下。

沉浸 就是让人专注在当前的目标（由设计者营造）情境下感到愉悦和满足，而忘记真实世界的情境。
移植到此处讲，就是尽可能隐藏掉手机的状态信息，使得用户完全沉浸在当前的情景中。

参考：[Android沉浸式状态栏]（http://www.jianshu.com/p/aca4fd6743b1）

使用：WindowManager.LayoutParams.FLAG_TRANSLUCENT_STATUS，Added in API level 19，如果这个flag被设置，View.SYSTEM_UI_FLAG_LAYOUT_STABLE 和 View.SYSTEM_UI_FLAG_LAYOUT_FULLSCREEN 这两个flag会被自动添加到system UI visibility中。

理解一下两点对于下面的阐述很重要：

  • 一旦这个Flag被设置，则状态栏就变透明，而且是被内容布局覆盖住
  • fitsSystemWindows = “true” 用于控制是否为状态栏留出空间

以下，我分三种情况进行阐述:

###1. 把状态栏用APP的主色调纯色覆盖
效果如图所示：

首先设置状态栏透明（半透明）

然后创建一个与statusBar等高的view，设置背景色,并添加到布局中，然后获取根布局，设置FitsSystemWindow和ClipToPadding为true。在添加上面的Flag时，会使得内容布局覆盖在statusBar上方，两者重叠，故使用FitsSystemWindow来调整出一个位置，使内容居于状态栏下方。

最后只需在activity中调用setStatusColor(activity,color)即可。

###2. 使用图片做背景，并是内容延伸至状态栏
效果如图所示：

首先，与上面的一样，添加状态栏透明Flag，实现了全屏效果，内容布局直接覆盖了statusBar,背景图片就自然地延伸到状态栏，再对根布局进行设置以空出statusBar的位置，使得内容布局居于状态栏下方。

3. 在DrawerLayout中应用

效果如图所示：

DrawerLayout布局包含两个字view,一个是内容，一个是抽屉。在这个布局文件中，根节点设置android:fitsSystemWindows="false"
内容主要包含一个ToolBar和一个FrameLayout，其中，ToolBar由一个AppBarLayout包裹

第一步还是添加FLAG_TRANSLUCENT_STATUS。
这里设置fitsSystemWindows="false",是因为这里不采用添加一个view的方式来实现，而是采用给AppBarLayout动态设置paddingTop,数值为状态栏的高度，这样ToolBar的上方就空出一块与状态栏等高的区域，且颜色也是相同的。

而抽屉那一块因为没有设置paddingTop,所以整个都移上去，因为抽屉布局都已经固定，所以不必担心抽屉上的内容跟状态栏重叠。

4. 最后

至此，基本完成通常情境下的需求。
