Android 4.4 以上沉浸式状态栏实现

这几天看了很多关于沉浸式状态栏的实现方案，总的在此汇总整理一下。

沉浸 就是让人专注在当前的目标（由设计者营造）情境下感到愉悦和满足，而忘记真实世界的情境。
移植到此处讲，就是尽可能隐藏掉手机的状态信息，使得用户完全沉浸在当前的情景中。

参考：Android沉浸式状态栏

使用：WindowManager.LayoutParams.FLAG_TRANSLUCENT_STATUS，Added in API level 19，如果这个flag被设置，View.SYSTEM_UI_FLAG_LAYOUT_STABLE 和 View.SYSTEM_UI_FLAG_LAYOUT_FULLSCREEN 这两个flag会被自动添加到system UI visibility中。

理解一下两点对于下面的阐述很重要：

  • 一旦这个Flag被设置，则状态栏就变透明，而且是被内容布局覆盖住
  • fitsSystemWindows = “true” 用于控制是否为状态栏留出空间
  • Android 4.4 以上沉浸式状态栏实现
      • 1. 把状态栏用APP的主色调纯色覆盖
      • 2. 使用图片做背景，并是内容延伸至状态栏
      • 3. 在DrawerLayout中应用
      • 4. 最后

以下，我分三种情况进行阐述:

1. 把状态栏用APP的主色调纯色覆盖

效果如图所示：

纯色状态栏

首先设置状态栏透明（半透明）

然后创建一个与statusBar等高的view，设置背景色,并添加到布局中，然后获取根布局，设置FitsSystemWindow和ClipToPadding为true。在添加上面的Flag时，会使得内容布局覆盖在statusBar上方，两者重叠，故使用FitsSystemWindow来调整出一个位置，使内容居于状态栏下方。

/**
* creat a view which is as high as status bar
**/
public static View createStatuView(Activity activity,int color){
        //获取状态栏高度
        int statusBarHeight = getStatusBarHeight(activity);

        View statusView = new View(activity);
        LinearLayout.LayoutParams params = new LinearLayout.LayoutParams(
        ViewGroup.LayoutParams.MATCH_PARENT,statusBarHeight);
        statusView.setLayoutParams(params);
        statusView.setBackgroundColor(color);
        return statusView;
    }
/**
 * 
**/
public static void setStatusColor(Activity activity,int color){
        if(Build.VERSION.SDK_INT<Build.VERSION_CODES.LOLLIPOP){
            //小于5.0，5.0及以上创建工程时就默认使用colorPrimary的颜色，故不必再作考虑
            //设置状态栏透明
            activity.getWindow().addFlags(WindowManager.LayoutParams.FLAG_TRANSLUCENT_STATUS);
            View statusView = createStatuView(activity,color);

            //添加statusview到布局中
            ViewGroup decorView = (ViewGroup) activity.getWindow().getDecorView();
            decorView.addView(statusView);

            //设置根布局
            ViewGroup rootView = (ViewGroup)((ViewGroup) activity
            .findViewById(android.R.id.content)).getChildAt(0);
            rootView.setFitsSystemWindows(true);
            rootView.setClipToPadding(true);
        }
    }
public static int getStatusBarHeight(Activity activity){
        int resourceId = activity.getResources()
        .getIdentifier("status_bar_height","dimen","android");
        int height =  activity.getResources().getDimensionPixelOffset(resourceId);
        Log.i("status bar ","height :"+height);
        return height;
    }

最后只需在activity中调用setStatusColor(activity,color)即可。

2. 使用图片做背景，并是内容延伸至状态栏

效果如图所示：

图片延伸至状态栏

首先，与上面的一样，添加状态栏透明Flag，实现了全屏效果，内容布局直接覆盖了statusBar,背景图片就自然地延伸到状态栏，再对根布局进行设置以空出statusBar的位置，使得内容布局居于状态栏下方。

this.getWindow().addFlags(WindowManager.LayoutParams.FLAG_TRANSLUCENT_STATUS);
        // setup rootview
        ViewGroup rootview = (ViewGroup) ((ViewGroup) findViewById(android.R.id.content)).getChildAt(0);
        rootview.setFitsSystemWindows(true);
        rootview.setClipToPadding(true);

3. 在DrawerLayout中应用

效果如图所示：

内容页

抽屉页

DrawerLayout布局包含两个字view,一个是内容，一个是抽屉。在这个布局文件中，根节点设置android:fitsSystemWindows="false"
内容主要包含一个ToolBar和一个FrameLayout，其中，ToolBar由一个AppBarLayout包裹

<?xml version="1.0" encoding="utf-8"?>
<android.support.design.widget.CoordinatorLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:fitsSystemWindows="false"
    tools:context="com.njrobot.huangyouqiang.immersive.DrawerActivity">

    <android.support.design.widget.AppBarLayout
        android:id="@+id/toolbarWrapper"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:theme="@style/AppTheme.AppBarOverlay">

        <android.support.v7.widget.Toolbar
            android:id="@+id/toolbar"
            android:layout_width="match_parent"
            android:layout_height="?attr/actionBarSize"
            android:background="?attr/colorPrimary"
            app:popupTheme="@style/AppTheme.PopupOverlay" />

    </android.support.design.widget.AppBarLayout>

    <include layout="@layout/content_drawer" />
</android.support.design.widget.CoordinatorLayout>

第一步还是添加FLAG_TRANSLUCENT_STATUS。
这里设置fitsSystemWindows="false",是因为这里不采用添加一个view的方式来实现，而是采用给AppBarLayout动态设置paddingTop,数值为状态栏的高度，这样ToolBar的上方就空出一块与状态栏等高的区域，且颜色也是相同的。

getWindow().addFlags(WindowManager.LayoutParams.FLAG_TRANSLUCENT_STATUS);
AppBarLayout appBarLayout = (AppBarLayout) findViewById(R.id.toolbarWrapper);
appBarLayout.setPadding(0,getStatusBarHeight(this),0,0);

而抽屉那一块因为没有设置paddingTop,所以整个都移上去，因为抽屉布局都已经固定，所以不必担心抽屉上的内容跟状态栏重叠。

4. 最后

至此，基本完成通常情境下的需求。

关于ANDROID的AIDL的使用

aidl 即 Android Interface Definition Language ,可用来定义Android进程间的通信接口。

1. 首先，有一个客户端和一个服务端。服务端中在src下创建一个**.aidl文件，并给出所需接口的说明，如下：

ADT会自动编译并在gen目录下生成一个对应的java文件：

2. 创建一个service实现该接口：

service对外提供的操作接口就是 AVChat.Stub。
>ps：当一个service可供别的进程访问时，需设置 ` android:exported= "true" `，如果intentFilter中的action至少有一个的话，默认 ` android:exported = "true" `,
否则，默认 `android :exported = "false" `。

3. 在客户端，编写一份同样的aidl文件在相同的包下，即包含一份相同的接口声明。

4. 在客户端的activity中绑定服务。

activity 中持有一个接口对象，在绑定服务端service成功后，被赋值
mAVChat = AVChat.Stub.asInterface(service); 其中的参数service是服务端service返回的IBinder对象。

5. 然后就可以通过接口对象调用服务了。因为这个服务是异步的，不保证一定在ui线程报ANR之前完成，尽量用多线程。

aidl 中如果要传送自定义类，需实现Parcelable接口。并为这个类写一个aidl文件
