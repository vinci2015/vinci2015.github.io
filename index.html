关于ANDROID的AIDL的使用

发表于 2016-07-25   |   分类于 android   |  

aidl 即 Android Interface Definition Language ,可用来定义Android进程间的通信接口。

1. 首先，有一个客户端和一个服务端。服务端中在src下创建一个**.aidl文件，并给出所需接口的说明，如下：

ADT会自动编译并在gen目录下生成一个对应的java文件：

2. 创建一个service实现该接口：

service对外提供的操作接口就是 AVChat.Stub。
>ps：当一个service可供别的进程访问时，需设置 ` android:exported= "true" `，如果intentFilter中的action至少有一个的话，默认 ` android:exported = "true" `,
否则，默认 `android :exported = "false" `。

3. 在客户端，编写一份同样的aidl文件在相同的包下，即包含一份相同的接口声明。

4. 在客户端的activity中绑定服务。

activity 中持有一个接口对象，在绑定服务端service成功后，被赋值
mAVChat = AVChat.Stub.asInterface(service); 其中的参数service是服务端service返回的IBinder对象。

5. 然后就可以通过接口对象调用服务了。因为这个服务是异步的，不保证一定在ui线程报ANR之前完成，尽量用多线程。
